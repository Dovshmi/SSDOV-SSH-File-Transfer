Main behavior now:
  - Normal SSH client TUI still opens with:
      ssh -p 2222 download@server

  - Direct download still works:
      ssh -p 2222 download@server 'download file.txt' > file.txt

  - Upload uses legacy SCP only:
      scp -O -P 2222 /local/file.jpg download@server:file.jpg

  - Server enables SCP upload with:
      ./ssdov -addr :2222 -user download -upload ~/Downloads

  - Server disables upload if upload is empty/off/none/disabled/no:
      ./ssdov -addr :2222 -root ~/Downloads -upload off

  - If -upload is set and -root is NOT explicitly set, browsing/download root defaults to the upload folder.

  - If both are set, explicit -root still wins:
      ./ssdov -addr :2222 -root /srv/files -user download -upload ~/Downloads

Server-side setup menu:
  - Running ./ssdov with no parameters ALWAYS opens a local Bubble Tea setup menu before the server starts.
  - Existing flag/env startup still works when any parameter is passed.
  - Setup fields:
      Port
      Download folder/root
      Upload folder
      SSH username
      Server password
      Start on boot
      Start server with these options
  - Setup menu displays detected LAN/local IP:
      Local IP: 192.168.1.183
  - Upload field behavior:
      Empty internally by default.
      Empty display shows: off (SCP upload disabled)
      If a folder is typed, display shows: <folder> (enables scp -O uploads)
      If user types off/none/disabled/no, upload is disabled.
  - Start on boot behavior:
      Default: disabled
      Press Enter or Space on the field to toggle enabled/disabled.
      Enabled display: enabled (install user startup service)

Folder creation prompt:
  - If download folder does not exist, setup prompts instead of failing silently:
      Folder does not exist: <path> (Download folder). Press Y to create it, N to edit.
  - If upload folder is enabled but missing, setup prompts:
      Folder does not exist: <path> (Upload folder). Press Y to create it, N to edit.
  - Pressing Y uses os.MkdirAll(path, 0755).
  - Pressing N returns cursor to the relevant folder field.

Startup service behavior:
  - If Start on boot is enabled in setup, SSDOV writes a user systemd service to:
      ~/.config/systemd/user/ssdov.service
  - Service file mode is set to 0600.
  - It then runs:
      systemctl --user daemon-reload
      systemctl --user enable ssdov.service
  - The service uses the currently selected setup options in ExecStart.
  - IMPORTANT SECURITY NOTE:
      Current implementation writes SSHDOWN_PASSWORD into the user service file if a password was entered.
      File mode is 0600, but the password is still saved on disk.
      Better future improvement: use authorized keys, an env-file, keyring, or ask password at start instead of writing password directly.

Local IP / client command behavior:
  - network.go has detectLocalIP(), choosing the first non-loopback IPv4 address from active interfaces.
  - Setup menu shows this local IP.
  - startServer logs copy-friendly client commands:
      Client TUI: ssh -p <port> <user>@<local-ip>
      Client upload: scp -O -P <port> local-file <user>@<local-ip>:filename
    Upload command log only appears if upload is enabled.

Security/limits:
  - SCP upload is disabled unless -upload <directory> is set or setup upload folder is filled.
  - SCP upload destination must be a plain filename only, not a path.
  - Upload refuses overwrite.
  - Recursive directory upload/download is supported with legacy `scp -O -r`.
  - Modern scp without -O is not supported because it uses SFTP by default.
  - No SFTP subsystem exists yet.

Important code details:

    - App has Root and UploadDir.
    - serverConfig includes Addr, Root, UploadDir, HostKey, User, Password, AuthKeys, InsecureAllowAny, InstallService.

    - flagServerConfig() preserves flag/env startup for scripts/systemd.

  service.go:
    - buildStartupService(cfg, binaryPath) creates the user systemd unit content.
    - writeStartupService(cfg, binaryPath) writes ~/.config/systemd/user/ssdov.service with mode 0600.
    - installStartupService(cfg) writes the unit, daemon-reloads, and enables it with systemctl --user.
    - systemdQuoteArg() quotes ExecStart/environment args.

  network.go:
    - detectLocalIP() returns first active non-loopback IPv4, or "unknown".

  tui.go:
